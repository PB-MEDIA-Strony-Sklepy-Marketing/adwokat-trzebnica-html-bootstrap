## Notes for Maintainers

When updating this changelog:
1. Always update the [Unreleased] section with new changes
2. When releasing a new version, move items from [Unreleased] to a new version section
3. Add a date in YYYY-MM-DD format to the version header
4. Update links at the bottom of the file
5. Follow the Keep a Changelog format strictly
6. Be concise but descriptive in change descriptions
7. Group changes by type (Added, Changed, etc.)
8. Reference issue numbers when applicable (e.g., "Fixed login bug (#123)")
